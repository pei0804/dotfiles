# デザインの 4 原則

ノンデザイナーズ・デザインブック (Robin Williams) の 4 原則。図解の質はほぼここで決まる。

## 1. 近接 (Proximity)

関連する要素は近づけ、関係のないものは離す。空間そのものに意味を持たせる。

### 実践

- 同じ役割の箱は隙間を狭くしてグルーピング
- グループ間は明確に大きな余白を空ける（グループ内余白の 2〜3 倍）
- 「どこからどこまでが 1 つの塊か」を視線で追えるようにする

### よくある失敗

- すべての要素を等間隔に並べる → グループが消える
- 「だいたい近い」程度の配置 → 読み手が迷う

### viewBox 1280x720 での目安

- グループ内の要素間隔: 20〜40px
- グループ間の余白: 80〜120px
- 全体の外側余白: 60px 以上

## 2. 整列 (Alignment)

すべての要素を意図的に揃える。中央揃え、左揃え、グリッドのいずれかを選び、図全体で一貫させる。

### 実践

- 軸を 2〜4 本程度に絞る（縦軸 1〜2 本、横軸 1〜2 本）
- すべての要素はそのいずれかの軸に揃える
- 「目立たせるためにあえてずらす」は最終手段。99% は揃える

### よくある失敗

- 中央揃えと左揃えが混在 → 全体が雑に見える
- ピクセル単位でズレている（120px と 118px みたいな） → 視覚的に違和感

### SVG 実装のコツ

- 主要要素の x 座標・y 座標は 10px 刻みで揃える
- 列・行を意識して、似た役割の要素は同じ x または y に置く

## 3. 反復 (Repetition)

同じ役割の要素は同じ見た目にする。色・形・サイズ・線の太さ・フォントを意識的に繰り返す。

### 実践

- 「サービス」を表す箱は全部同じ寸法・同じ枠線・同じ角丸にする
- タイトル・サブタイトル・本文・キャプションの 4 段階くらいのテキストスタイルを決め、それを通す
- 矢印の太さ・矢じりの形は、意味が変わらない限り統一する

### よくある失敗

- 似た役割の箱が、ちょっとずつ大きさが違う → 「違うもの」に見える
- 同じ意味の矢印が、たまたま色・太さが違う → 「違う意味があるのか？」と読み手が悩む

### チェックポイント

- すべての箱の角丸 (rx) は揃っているか？
- すべての矢印の stroke-width は揃っているか？（意味が違うなら違ってよい）
- フォントサイズは 3〜4 段階以内に収まっているか？

## 4. コントラスト (Contrast)

重要なものを目立たせる。差は「大きく」つける。中途半端な差は混乱を生む。

### 実践

- タイトルは本文の 2 倍以上のサイズで
- 強調したい色は、他の要素と明確に違う色相を選ぶ（同系色での濃淡は弱い）
- 太字 / 細字、塗りつぶし / 枠線、大 / 小、明 / 暗 を組み合わせる

### よくある失敗

- 「ちょっと違う色」「ちょっと大きい」 → 読み手は違いに気づかない
- 全要素を強調 → 何も強調されない（強調の希釈）

### コントラスト比（アクセシビリティ）

WCAG AA: 通常テキスト 4.5:1、大きいテキスト 3:1 以上。

確認方法: ブラウザ開発者ツールや https://webaim.org/resources/contrastchecker/ で確認できる。

## 4 原則の自己チェック（出力前）

- [ ] **近接**: グループ内は近く、グループ間は離れているか？
- [ ] **整列**: すべての要素が明確な軸に揃っているか？
- [ ] **反復**: 同じ役割のものは同じ見た目になっているか？
- [ ] **コントラスト**: 重要度の差ははっきり大きくつけているか？

4 つすべてが効いていれば、よほどのことがない限り「読める」図になる。

## レイアウト健全性チェック（出力前・別軸）

4 原則が効いていても、技術的なレイアウト崩れがあると読めない。出力前に以下を必ず確認する。

### テキストと図形の関係

- [ ] **矢印・線の上にテキストが重なっていないか？**
  - 矢印にラベルを置くときは、矢印を回避する位置（横にずらす）か、矢印を分割して間に置く。矢印の上に直接置かない
  - SVG の `<text>` は `y` 属性が **baseline** 位置を示す。フォントサイズ 24px の文字を矢印の上に置く場合、文字の上端は `y - 24` まで延びる。矢印 y との差が「フォントサイズ + 余白 8px」以上ないと文字の上半分が矢印にかぶる
  - 具体的な計算: 矢印が `y=320` にあり、ラベルが上に来るなら、ラベルの `text y` は `<= 320 - fontSize - 8 = 320 - 24 - 8 = 288` 以下。下に置くなら `>= 320 + 8 = 328` 以上
  - **矢印が太いとき (`stroke-width >= 6px`) は余白に矢印太さの半分を加算する**: 矢印 `y=320` で `stroke-width=10` なら矢印は y=315 〜 y=325 の範囲を占める。上ラベルは `<= 315 - fontSize - 8`、下ラベルは `>= 325 + 8` で計算する
  - 「矢印にラベルを乗せていない」と思い込むのではなく、座標値で実際に検算する
  - SVG では描画順が後のものが上に来る。テキストを後に描画してもアンチエイリアスで重なりは目立つ
- [ ] **枠内のテキストが枠の境界を超えていないか？**
  - 日本語テキストの幅は概ね `フォントサイズ × 文字数`（半角は半分）。枠の幅と比較して足りなければ縮める
  - 文字数が多いときは複数行に分けるか（`<tspan>` 利用）、フォントサイズを下げるか、枠を広げる
  - パディング（枠の内側余白）は最低でも `フォントサイズ × 0.5` 確保する
- [ ] **テキスト同士が重なっていないか？**
  - 隣接するラベルが近すぎると重なる。要素間隔を見直す

### 矢印の太さ

- [ ] **矢印の線幅 (stroke-width) は意味伝達に必要な範囲か？**
  - 通常: 2〜3px
  - 強調が必要なフロー: 4〜6px
  - これを超えると（8px 以上）「装飾的な太さ」になり図全体のバランスを崩す
- [ ] **強調は太さ以外の手段で行えないか？**
  - 色を変える（メインフローだけ目立つ色）
  - 位置を変える（中央に置く）
  - サイズを変える（要素自体を大きく）
  - これらの方が太さで強調するより上品で、4 原則のコントラストとも整合する
