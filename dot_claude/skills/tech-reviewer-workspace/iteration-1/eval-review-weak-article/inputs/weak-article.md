# Docker Composeでの開発環境構築

## はじめに

Docker Composeを使うと、開発環境を簡単に構築できます。複数のコンテナをまとめて管理できるので便利です。

## Docker Composeとは

Docker Composeは、複数のDockerコンテナを定義・管理するためのツールです。YAMLファイルにサービスを記述することで、コマンド一つで環境を立ち上げることができます。

## セットアップ方法

Docker Composeを使うには、まずDockerをインストールします。その後、`docker-compose.yml`というファイルを作成し、必要なサービスを定義します。

サービスには、Webアプリケーション、データベース、キャッシュサーバーなどを含めることができます。それぞれのサービスにはイメージやポート、環境変数などを設定します。

設定が完了したら、`docker compose up`コマンドで環境を起動します。

## ネットワーク

Docker Composeでは、サービス間の通信のためにネットワークが自動的に作成されます。サービス名をホスト名として使用できるので、接続設定が簡単です。

## ボリューム

データの永続化にはボリュームを使います。ボリュームを設定することで、コンテナを削除してもデータが保持されます。

## 環境変数

環境変数を使うことで、設定値を柔軟に管理できます。`.env`ファイルを使う方法もあります。

## まとめ

Docker Composeを使えば、開発環境を効率的に構築・管理できます。チーム全体で同じ環境を共有できるため、「自分の環境では動く」問題を解消できます。ぜひ活用してみてください。
