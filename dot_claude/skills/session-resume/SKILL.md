# Session Resume

中断したClaude Codeセッションのコンテキストを復元し、作業を即座に再開するためのスキル。

## transcript-recallとの違い

- **transcript-recall**: 過去の会話を閲覧・振り返る（読むだけ）
- **session-resume**: 作業コンテキストを復元して続行する（アクション志向）

## 手順

### 1. 最近のセッション一覧を取得

transcript-recallのパーサーを使って一覧を取得する。

```bash
~/.claude/skills/transcript-recall/scripts/recall.sh --list
```

特定プロジェクトに絞る場合:
```bash
~/.claude/skills/transcript-recall/scripts/recall.sh --list --project vision
```

### 2. ユーザーにどのセッションを再開したいか確認

一覧を見せて選んでもらう。「さっきの」と言われた場合は直近のセッションを選ぶ。同じプロジェクトのセッションが複数ある場合は日時で確認する。

### 3. トランスクリプトを読み込む

```bash
~/.claude/skills/transcript-recall/scripts/recall.sh --file /path/to/transcript.jsonl
```

### 4. コンテキスト復元サマリーを生成

トランスクリプトから以下を抽出・推測してまとめる:

```
## セッション復元

**プロジェクト**: project-name
**作業ディレクトリ**: /path/to/project
**日時**: YYYY-MM-DD HH:MM

### 何をやっていたか
- ユーザーの目的・ゴールを簡潔に
- 実施済みの作業ステップ

### 完了したこと
- 完了したタスクのリスト
- 作成・変更したファイル

### 未完了・残タスク
- まだ終わっていない作業
- ユーザーが最後にリクエストしていたこと
- エラーや問題が残っていた場合はその内容

### 作業ブランチ・状態
- gitブランチ名（わかれば）
- 最後のコミットメッセージ（わかれば）
```

### 5. 現在の状態を確認

サマリーを出力した後、実際に作業ディレクトリに移動して現在の状態を確認する:

- `git status` でブランチと変更状態
- `git log --oneline -5` で最近のコミット
- 未完了タスクに関連するファイルが存在するか

これにより、セッション終了後に手動で変更があった場合も把握できる。

### 6. ユーザーに確認して作業続行

「この状態から続けますか？」と確認し、了承を得たら未完了タスクから作業を再開する。

## ポイント

- トランスクリプトの最後の方のメッセージが最も重要（中断直前の状態）
- ユーザーの最後の発言が「次にやりたかったこと」のヒントになることが多い
- tool_useの結果（ファイル編集、コマンド実行）から具体的な変更内容を把握する
- 会話全体を再現するのではなく、続行に必要な情報だけを凝縮する
